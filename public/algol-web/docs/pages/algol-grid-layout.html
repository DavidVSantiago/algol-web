<algol-grid-layout>

Visão Geral

O algol-grid-layout é o componente fundamental de diagramação da Algol UI. Ele estabelece uma malha bidimensional (CSS Grid) que gerencia o posicionamento, o espaçamento e o alinhamento de seus componentes filhos.

Regra de Composição: Para obter resultados previsíveis e suporte total às funcionalidades de span (expansão) e responsividade da biblioteca, este componente deve ser utilizado em conjunto com os elementos <algol-grid-item> e/ou <algol-flex-item> como seus filhos diretos. 
<!-- Exemplo de uso composto -->
<algol-grid-layout cols="repeat(4, 1fr)" gap="1.5">
    <algol-grid-item colspan="3">Área Principal</algol-grid-item>
    <algol-flex-item>Barra Lateral</algol-flex-item>
</algol-grid-layout>

Sinergia entre Componentes

  • Com <algol-grid-item>: Indicado para controle cirúrgico de posição e expansão na grade, além de suportar overlays de imagem e alinhamentos individuais.
  • Com <algol-flex-item>: Ideal quando o conteúdo dentro da célula da grade também precisa de comportamento flexível (flex-wrap, distribuição de espaço) ou quando atua como um contêiner simples de expansão.

Configuração de Colunas e Espaçamento

Atributo Mapeamento CSS Descrição
cols grid-template-columns Define a malha de colunas. Aceita valores CSS como 1fr 2fr ou repeat(3, 1fr).
gap gap Espaçamento entre as linhas e colunas da grade.
padding padding Espaçamento interno do contêiner. Números são convertidos automaticamente para vw.
color background-color Define a cor de fundo do layout de grade.

Alinhamento e Responsividade

O layout centraliza os itens por padrão (center), mas permite customização total para os estados Desktop e Mobile:

Propriedade Desktop/Tablet Mobile (Break)
Estrutura de Colunas cols colsbreak
Espaçamento (Gap) gap gapbreak
Alinhamento Horizontal posh poshbreak
Alinhamento Vertical posv posvbreak

Comportamento Interno

Desempenho: O algol-grid-layout utiliza o método estático injectStyles() para garantir que as regras CSS necessárias para a responsividade e variáveis sejam injetadas apenas uma vez no <head> do documento, otimizando a renderização.
