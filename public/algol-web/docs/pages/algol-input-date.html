<algol-input-date>

Visão Geral

Componente estendido para seleção de datas. Ele aciona o seletor de calendário nativo do sistema operacional/navegador e possui validação inteligente de intervalos de tempo.

<algol-input-date label="Data de Nascimento" min="01/01/1900" max="31/12/2026"></algol-input-date>

Atributos de Configuração

Herda as propriedades base de inputs (label, disabled, required, value) e adiciona controle de limites de data:

Atributo Descrição Valores Aceitos
min A data mais antiga permitida para seleção. Data (Aceita formato universal YYYY-MM-DD ou brasileiro DD/MM/YYYY)
max A data limite no futuro permitida para seleção. Data (Aceita formato universal YYYY-MM-DD ou brasileiro DD/MM/YYYY)

Mecânicas Específicas

Conversor Automático de Formato: Os navegadores exigem estritamente o formato ISO (YYYY-MM-DD) para propriedades nativas de data. Para facilitar o uso no Brasil, a Algol UI intercepta valores em DD/MM/YYYY passados no min e max e faz a tradução interna automaticamente antes de repassar ao DOM.
Filtro de Evento (Otimização): Diferente do input de texto padrão, o algol-input-date tem a diretiva _canEmitInput = false. Isso significa que ele não emite o evento de tempo real a cada clique dentro do calendário nativo, garantindo que você só receba o dado quando a data estiver completamente selecionada.

Eventos Emitidos

Devido à supressão intencional do evento em tempo real, o monitoramento deve ser feito exclusivamente através da confirmação do valor:

  • algol-change: Disparado quando o usuário termina de escolher a data no calendário ou finaliza a digitação manual e o campo perde o foco. 
    document.querySelector('algol-input-date').addEventListener('algol-change', e => {
  const { oldValue, value } = e.detail;
  // O value virá sempre no formato padrão AAAA-MM-DD
  console.log(`Data confirmada: ${value}`); 
});

Validação Interna

Este componente trabalha de mãos dadas com a API ElementInternals. Se o usuário burlar o calendário (digitando manualmente) e inserir uma data menor que o min, o formulário identificará o erro rangeUnderflow. Se for maior que o max, acusará rangeOverflow, travando a submissão.
