<algol-switch-radio>

Visão Geral

Componente de opção única com interface de interruptor (Toggle). Ele força o usuário a escolher apenas uma opção de um grupo, mas apresenta a escolha como chaves deslizantes, conferindo um aspecto mais tecnológico e imediato à interface.

<algol-switch-radio name="tema" value="claro" checked>Modo Claro</algol-switch-radio>
<algol-switch-radio name="tema" value="escuro">Modo Escuro</algol-switch-radio>

Uso Semântico Recomendado

Quando utilizar? Ao contrário de listas longas onde o <algol-radio> tradicional brilha, o <algol-switch-radio> é perfeito para escolhas binárias de interface. Exemplos clássicos: alternar entre visualização em "Lista" ou "Grade", Modo "Claro" ou "Escuro", Pagamento "Mensal" ou "Anual".

Agrupamento Obrigatório

A Regra do name: Assim como seu irmão padrão (algol-radio), este componente depende inteiramente do atributo name para funcionar. Todos os switches que pertencem à mesma escolha devem ter o exato mesmo name para que a Algol UI consiga desligar um quando o outro for ligado.

Atributos de Configuração

Herdando as capacidades do BaseCheckable, ele suporta:

  • name: Obrigatório. Agrupa os switches concorrentes.
  • value: O valor único que esta chave representa no formulário.
  • checked: Faz com que esta opção inicie ligada.
  • disabled: Bloqueia o clique, travando a chave na posição atual.
  • required: Exige validação de preenchimento para o grupo no formulário.

Eventos Emitidos

Para reagir instantaneamente à troca de visualização (por exemplo, aplicar o modo escuro na hora), escute o evento algol-input no grupo:

  • algol-input: Disparado assim que a chave assume a posição "Ligado". 
    document.querySelectorAll('algol-switch-radio[name="tema"]').forEach(switchRadio => {
  switchRadio.addEventListener('algol-input', e => {
      const { value } = e.detail;
      console.log(`Tema alterado para: ${value}`);
      // Lógica para aplicar o tema na tela
  });
});
