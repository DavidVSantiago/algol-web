<algol-input-file>

Visão Geral

Componente avançado para upload de arquivos. Substitui a interface truncada e não-estilizável do navegador por um design elegante composto de campo de leitura + botão de ação. Possui gestão automática de FormData e filtros de formato estritos.

<algol-input-file name="documento" label="Anexar RG" text="Procurar..." accept=".pdf, .jpg"></algol-input-file>

Atributos de Configuração

Atributo Descrição Valores Aceitos
text O texto exibido dentro do botão de ação. Texto (Padrão: Search)
accept Filtro de segurança que restringe os tipos de arquivos que o usuário pode selecionar. MIME types ou Extensões (ex: image/png, .pdf, video/*)
multiple Permite a seleção de múltiplos arquivos simultaneamente. Na interface, os nomes serão separados por " | ". Booleano (presença do atributo)
name Nome da chave de envio do arquivo no formulário. Fundamental para o backend. Texto (Padrão fallback: file)
placeholder Texto de instrução no campo falso de texto. Texto (Padrão: Select file...)
disabled Desativa o botão, o campo de leitura e bloqueia a janela de seleção. Booleano (presença do atributo)

Motor de Dados e UX

Auto-injeção de FormData: Em formulários tradicionais, o envio de arquivos requer codificação complexa (multipart/form-data). O algol-input-file resolve isso nos bastidores: ao selecionar os arquivos, ele os empacota nativamente em um objeto FormData usando a chave definida no atributo name. Se o componente estiver dentro de uma tag <form>, ele já envia os arquivos binários corretos automaticamente via ElementInternals.
Validação Estrita (CustomError): Se o usuário arrastar um arquivo inválido (burlando a janela do sistema operacional) que não respeite a lista do accept, o componente rejeita a ação internamente, limpa a seleção e trava o formulário reportando um customError, impedindo uploads de arquivos indevidos no seu servidor.

Acesso aos Arquivos via JavaScript

Caso você não esteja usando submissão nativa de formulários e precise acessar os arquivos binários (Blob/File) para fazer um upload via fetch ou axios, o componente expõe a propriedade files:

const arquivos = document.querySelector('algol-input-file').files;
console.log("Objeto FileList contendo os binários:", arquivos);

Eventos Emitidos

  • algol-change: Disparado quando a janela do sistema operacional é fechada com uma seleção válida. O evento retorna os nomes dos arquivos em forma de texto (para uso em logs e interface). Para o binário, acesse a propriedade `.files` como mostrado acima. 
    document.querySelector('algol-input-file').addEventListener('algol-change', e => {
  const { oldValue, value } = e.detail;
  console.log(`Arquivos anteriores: [${oldValue}]. Novos arquivos: [${value}]`);
});
