BaseLayout

Visão Geral

O BaseLayout é a classe abstrata (não instanciada diretamente) que serve de fundação para componentes voltados puramente para diagramação, espaçamento e fluxo da página (como algol-grid-layout e algol-flex-item).

Seu objetivo é ser extremamente leve e focado em traduzir atributos do HTML diretamente para estilos (variáveis CSS), além de monitorar alterações na árvore do DOM caso os nós filhos mudem dinamicamente.

Tradução Avançada de CSS (ATTR_MAP)

Injeção com Prefixos/Sufixos: Nos componentes visuais interativos, a propriedade ATTR_MAP costuma mapear um atributo HTML diretamente para uma variável CSS de forma simples (ex: color="red" vira --color: red). O BaseLayout eleva isso. Ele entende objetos complexos de mapeamento, permitindo concatenar regras CSS vitais.
Exemplo interno: Se o componente herdeiro mapear colspan: { var: '--grid-col', prefix: 'span ' }, escrever colspan="2" no HTML será traduzido automaticamente para --grid-col: span 2 no CSS.

Eventos Emitidos (Mutation Observer)

Componentes de layout geralmente contêm dezenas de outros elementos filhos (textos, imagens, inputs). O BaseLayout inicia um "olheiro" (MutationObserver) para avisar quando a estrutura interna dele (o Light DOM) sofre adições ou remoções de elementos (útil para recálculos matemáticos de grid ou renderizações de frameworks). Isso é encapsulado em um único evento:

  • algol-light-dom-update: Disparado toda vez que um elemento "filho" é inserido ou deletado de dentro da tag do componente de layout. 
    componenteDeLayout.addEventListener('algol-light-dom-update', e => {
  const { origin, mutations } = e.detail;
  console.log("Os filhos deste layout mudaram!", origin);
  console.log("Detalhes da mutação do DOM:", mutations);
});

Injeção de Estilos no Documento

Desempenho: Esta classe possui o método estático _injectStyleOnHead. Como componentes de layout são muito replicados em uma página, eles costumam não usar Shadow DOM para não "esconder" os filhos do CSS global. Em vez disso, eles usam este método da BaseLayout para injetar suas regras CSS (medias queries e classes) apenas uma única vez na tag <head> do documento principal.
